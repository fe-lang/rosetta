# Target Contracts

Contracts identified as good candidates for Solidity → Fe side-by-side comparison, organized by niche. Priority informed by Ahmed's [Fe Use Cases](https://shade-octave-2bc.notion.site/Fe-Use-Cases-2bcf63ce63cf800d8628fbf1d4f40e67) and [Content & Comms Strategy](https://shade-octave-2bc.notion.site/Fe-Content-Comms-Strategy-2ccf63ce63cf8028a7bef58cdc97e06f).

## Cryptography

### ✅ Merkle Trees (done — from fe-zkkit)
- **LeanIMT + Sparse Merkle Tree** from [zk-kit](https://github.com/privacy-scaling-explorations/zk-kit)
- Fe wins 3–15% on gas with traits (`Hasher`), const generics (`DEPTH`), and zero-cost abstractions
- Source: Grant's [fe-support branch](https://github.com/g-r-a-n-t/zk-kit/tree/fe-support/fe-zkkit)

### 🚧 ZK Verifiers
- **Groth16 Pairing library** from [Tornado Cash Verifier.sol](https://github.com/tornadocash/tornado-core/blob/master/contracts/Verifier.sol) — EC precompile calls (ecAdd, ecMul, ecPairing), all assembly
- **BN254.sol** from [EigenLayer](https://github.com/Layr-Labs/eigenlayer-contracts/blob/dev/src/contracts/libraries/BN254.sol) ($9.1B TVL) — comprehensive EC library
- **SemaphoreVerifier.sol** — snarkJS-generated Groth16 verifier, 100% assembly
- Prior art: [fe-crypto verifiers](https://github.com/g-r-a-n-t/fe-crypto/tree/main/verifiers) (Fe v1)

### Planned: Poseidon Hash
- **poseidon_gencontract.js** from [iden3/circomlibjs](https://github.com/iden3/circomlibjs/blob/main/src/poseidon_gencontract.js) — *can't even be written in Solidity*, generated as raw bytecode by JavaScript
- The ultimate Fe showcase: a hash function expressed as readable code vs machine-generated bytecode
- Used by MACI, Semaphore, and other privacy protocols

## DeFi Math

### 🚧 Core Math Libraries
- **FullMath.sol** from [Uniswap V3](https://github.com/Uniswap/v3-core/blob/main/contracts/libraries/FullMath.sol) ($1.7B TVL) — 512-bit mulDiv via CRT + Newton-Raphson, 100% assembly. ~117 lines. **The "hello world" of why Fe matters.**
- **WadRayMath.sol** from [Aave V3](https://github.com/aave/aave-v3-core/blob/master/contracts/protocol/libraries/math/WadRayMath.sol) ($25B TVL) — fixed-point wad/ray arithmetic, all assembly. ~120 lines.
- **MathLib.sol + SharesMathLib.sol** from [Morpho Blue](https://github.com/morpho-org/morpho-blue/blob/main/src/libraries/MathLib.sol) ($6.7B TVL) — minimal, clean. Already near-clean Solidity, shows Fe is better even without the assembly crutch.

### Planned: Advanced Math
- **TickMath.sol** from Uniswap V3 — `sqrt(1.0001^tick)` with magic constants
- **BitMath.sol** from Uniswap V4 — De Bruijn MSB/LSB, all assembly
- **FixedPointMathLib.sol** from [Solady](https://github.com/Vectorized/solady/blob/main/src/utils/FixedPointMathLib.sol) — cherry-pick sqrt, exp2, mulWad (the maximum-assembly comparison)

## Contract Upgradability

### Planned: Proxy Patterns
- **Diamond.sol + LibDiamond.sol** from [EIP-2535 reference](https://github.com/mudgen/diamond-3-hardhat/tree/main/contracts) — delegatecall routing, storage slots
- **Proxy.sol + ERC1967Utils.sol** from [OpenZeppelin](https://github.com/OpenZeppelin/openzeppelin-contracts/tree/master/contracts/proxy) — canonical proxy, ~30 lines of pure assembly
- **SafeProxy.sol** from [Safe](https://github.com/safe-global/safe-smart-account/blob/main/contracts/proxies/SafeProxy.sol) — minimal proxy with assembly bit-shifting

## On-chain Gaming

### Planned: MUD Components
- **ResourceId.sol** from [MUD](https://github.com/latticexyz/mud/blob/main/packages/store/src/ResourceId.sol) — user-defined value type, bit-level encoding
- **FieldLayout.sol + Schema.sol** — ECS metadata packing into bytes32
- **Storage.sol + Memory.sol** — raw storage/memory with arbitrary byte offsets
- Prior art: [fe-mud](https://github.com/g-r-a-n-t/fe-mud)

## MEV / Arbitrage

### Planned
- **FlashBotsMultiCall** from [simple-arbitrage](https://github.com/flashbots/simple-arbitrage/blob/main/contracts/BundleExecutor.sol) — canonical MEV pattern, ~70 lines
- Note: real MEV bots use Huff or raw bytecode (competitive advantage). Fe's value here is offering a readable alternative with comparable gas.
